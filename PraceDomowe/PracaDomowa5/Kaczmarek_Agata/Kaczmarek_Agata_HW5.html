Partial Dependence Profiles (PDP)

Poniej dla wybranych zmiennych z mojego zbioru policz PDP:

pdp_2 <- model_profile(explainer_ranger, variables=c("back_camera_mpix", "battery_mAh", "ram_gb", "diag"))
pdp_svm_2 <- model_profile(explainer_svm, variables=c("back_camera_mpix", "battery_mAh", "ram_gb", "diag"))

pdp_2$agr_profiles$`_label_` <- "ranger"
pdp_svm_2$agr_profiles$`_label_` <- "svm"
plot(pdp_2, pdp_svm_2)

plot(pdp_2, geom = "profiles")

plot(pdp_svm_2, geom = "profiles")

Jak wida powyej PDP dla obu modeli wyglda zupenie rnie - dla svm nie ma wyranich zmian, s one bardzo agodne, w przeciwiestwie do tych dla rangera. Najwiksze rnice pomidzy modelami mona dostrzec pomidzy wykresami dla battery_mAH oraz diag. Dla wykresu zbiorczego moemy rwnie zaobserwowa, e dla svm wszystkie linie s agodne, a dla rangera wiele jest jeszcze bardziej zmiennych (pofalowanych, szpiczastych) ni poprzednio.

Jeli popatrzymy szczegowo na wykresy dla modelu ranger to mona zauway znaczne zmiany w predykcjach w zalenoci o wzrostu parametru szczeglnie dla wikszoci badanych parametrw. Np wraz ze wzrostem ram_gb znacznie ronie przewidywana cena, co jest raczej rzeczywistym zaoeniem. Co ciekawe jednak zwikszanie wartoci back_camera_mpix nie powoduje wzrostu wartoci predykcji. Moemy zauway rwnie ciekawy spadek dla przektnej w okolicy 5.5-6.5.

Rne kombinacje rozmiaru siatki oraz rozkadu jej punktw. Wpyw na krzywe.

Zobaczmy jak na wybrane z wyej uzyskanych wyniwi wpynie zmiana rozmiaru siatki oraz zmiana rozkadu punktw.

pdp_2_s1 <- model_profile(explainer_ranger, variables="back_camera_mpix", N=1000, grid_points=10)
pdp_2_s2 <- model_profile(explainer_ranger, variables="back_camera_mpix", N=1000, grid_points=100)
pdp_2_s3 <- model_profile(explainer_ranger, variables="back_camera_mpix", N=100, grid_points=100)

pdp_2_s1$agr_profiles$`_label_` <- "N=1000, grid_points=10"
pdp_2_s2$agr_profiles$`_label_` <- "N=1000, grid_points=100"
pdp_2_s3$agr_profiles$`_label_` <- "N=100, grid_points=100"
plot(pdp_2_s1, pdp_2_s2, pdp_2_s3)

pdp_2_s4 <- model_profile(explainer_svm, variables="back_camera_mpix", N=1000, grid_points=10)
pdp_2_s5 <- model_profile(explainer_svm, variables="back_camera_mpix", N=1000, grid_points=100)
pdp_2_s6 <- model_profile(explainer_svm, variables="back_camera_mpix", N=100, grid_points=100)
pdp_2_s4$agr_profiles$`_label_` <- "N=1000, grid_points=10"
pdp_2_s5$agr_profiles$`_label_` <- "N=1000, grid_points=100"
pdp_2_s6$agr_profiles$`_label_` <- "N=100, grid_points=100"
plot(pdp_2_s1, pdp_2_s2, pdp_2_s3)

Jaki miao to wpyw na krzywe?

Zmniejszona liczba grid_points powodowaa mniej punktw zmian na krzywej, bliej jej byo do linii prostej ni pozostaym przypadkom, bya to prostsza amana. Zmiana N nie wpyna bardzo znaczco na wyniki, dla obu modeli wykresy niebieski i zielony s do siebie bardzo zblione. Nieznacznie wiksz redni predykcj uzyskujemy jednak dla mniejszego N.
